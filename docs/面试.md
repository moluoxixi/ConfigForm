# ConfigForm 面试口播稿

## 开场（30 秒）
大家好，我负责的是 ConfigForm，一个基于 Formily 思想重构的“配置化表单引擎”。它的目标是跨框架（React 18 / Vue 3）复用同一套表单领域模型与 Schema 能力，同时把 UI 适配层拆分出来，让业务能自由选择 Ant Design、Element Plus 或自研基础组件。

## 架构分层（1 分钟）
整体是四层 + 插件层的结构，自下而上讲会更清晰：
- 核心层（@moluoxixi/core）：框架无关的表单领域模型。包含 Form/Field/ArrayField 等模型、Schema 编译与合并、$ref/模板解析、验证引擎、联动引擎、事件系统、数据源管理等。
- 响应式层（reactive-react / reactive-vue）：适配器模式。Vue 用 @vue/reactivity，React 用 MobX，通过 setReactiveAdapter 在运行时切换。
- 框架层（@moluoxixi/react / @moluoxixi/vue）：只做 Schema 递归渲染与上下文注入，不做任何 UI 具体渲染。核心组件是 SchemaField / RecursionField，外加 useForm / useField / useSchemaItems 等通用 Hook。
- UI 层（ui-basic / ui-antd / ui-element-plus）：真正渲染 UI。ConfigForm、ArrayField、LayoutTabs 等组件在这里落地，并且用对应 UI 库的 Tabs/Collapse/Table/Button 渲染。
- 插件层（plugin-*）：i18n、JSON Schema 适配、低代码设计器核心、DevTools 等都作为插件，统一通过 ConfigForm + SchemaField 渲染，禁止在插件入口直接 JSX/h 直出 UI。

## 核心机制与流程（1~2 分钟）
我会按“从 schema 到渲染”来讲：
1. 入口是 ConfigForm 接收 schema。核心层先做 compile/merge/$ref/模板解析，校验 schema 的结构与组件映射。
2. 框架层用 SchemaField/RecursionField 递归渲染，通过上下文把 Form/Field 的状态注入到组件里。
3. 联动引擎 ReactionEngine 负责 watch 依赖、表达式求值与状态更新。它内置依赖图、循环检测、微任务批次计数防死循环，并支持数组上下文（$record/$index）与追踪器用于调试。
4. 验证引擎支持同步/异步校验、格式化规则与跨字段验证，结果统一回流到 Field 状态。
5. 数据源模块支持 request adapter、TTL 缓存、并发请求去重、AbortSignal 取消，并支持参数从 $values 解析路径引用。
6. 插件系统通过 Hook 管线（洋葱模型）串联 submit/validate/setValues/createField/reset，保证插件可以“拦截 + 增强”而不侵入核心流程。

## 关键设计思想（强调我做了什么取舍）
- 配置集中：表单配置统一放在 schema.decoratorProps，避免表单实例与组件 props 分散带来的维护成本。
- 适配器模式：用响应式适配器实现跨框架复用，把“状态管理”与“渲染框架”解耦。
- 插件化：核心层只暴露稳定能力，所有扩展通过插件接入，降低核心复杂度并保持可测试性。
- Schema 感知布局：LayoutTabs/Collapse/Steps 通过 useSchemaItems 获取面板信息，UI 组件只负责渲染，不重复逻辑。

## 关键注意点/踩坑（面试时能体现工程经验）
- 必须在运行前 setReactiveAdapter，否则 Form/Field 不会进入正确的响应式体系。
- 插件的 UI 一律通过 ConfigForm/SchemaField 渲染，不能直出 JSX/h，否则会破坏框架层与 UI 层的边界。
- 表达式引擎默认用 new Function 编译。若 schema 来自非可信输入，需要调用 setUnsafeExpressionEvalEnabled(false)。
- 联动要避免循环依赖与互相写值；虽然有循环检测与批次上限保护，但最佳实践是缩小 watch 范围并适度 debounce。
- 数据源有 TTL 缓存与去重逻辑；带 AbortSignal 的请求不会去重，以保证“独立取消”语义。
- 组件映射必须注册完整（内置组件 + 自定义组件），否则 schema 渲染时会出现缺失组件报错。

## 与 Formily 的差异（30 秒，可选）
- 响应式：Formily 是自研统一实现，这里用适配器（Vue 原生 + React/MobX）。
- 入口：ConfigForm 单组件开箱即用，表单动作按钮自动渲染。
- 配置：更集中在 decoratorProps，减少散落配置。
- 布局：内置 grid/inline/vertical，不依赖额外 FormLayout 组件。

## 结尾（10 秒）
总结一下，这个项目的核心价值是“跨框架复用 + 插件化扩展 + Schema 驱动的高一致性”。后续我也可以继续讲 DevTools、低代码设计器、JSON Schema 适配等插件的落地细节。
